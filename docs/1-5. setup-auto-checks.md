# 🧠 자동 검사 & 커밋 규칙 설정 가이드

## 🎯 목적
프로젝트 내 코드 품질을 자동으로 검사하고,
팀원들이 실수로 규칙을 어기지 않도록 방지하는 시스템을 구축합니다.

---

## ⚙️ 1️⃣ Commitlint
- 커밋 메시지가 `type(scope): subject` 형식을 따르지 않으면 차단됩니다.
- 예시:
  - ✅ `feat(auth): 로그인 기능 추가`
  - ❌ `add login`

**설정 파일:** `commitlint.config.js`  
**훅 파일:** `.husky/commit-msg`

---

## ⚙️ 2️⃣ Husky (Git Hook Manager)
- 커밋 직전에 자동으로 Lint 또는 Commitlint를 실행합니다.
- 규칙 위반 시 커밋이 취소됩니다.


---

## ⚙️ 3️⃣ CI (GitHub Actions)
- PR이 생성되면 GitHub이 자동으로 아래 검사 수행:
  1. **Lint** – 문법 및 스타일 오류 확인  
  2. **Test** – 단위 테스트 실행  
- 모든 검사 통과 시에만 병합 가능

**설정 파일:** `.github/workflows/ci.yml`

---

## 💡 실행 순서 요약
```
로컬 커밋 전 → Husky pre-commit 실행 → Lint 검사
커밋 메시지 입력 후 → Husky commit-msg 실행 → Commitlint 검사
PR 생성 시 → GitHub Actions 실행 → Lint + Test 검사
```

---

## ✅ 결과
- 코드 스타일과 커밋 메시지 자동 검증  
- 팀원 실수 방지 및 코드 품질 보장  
- “통과된 코드만 develop으로 병합”되는 안전한 환경 완성

## 📘 추가 설치 명령어 (로컬 개발자용)
```
npm install -D husky @commitlint/{config-conventional,cli}
npx husky install
npx husky add .husky/commit-msg "npx --no -- commitlint --edit \$1"
```
