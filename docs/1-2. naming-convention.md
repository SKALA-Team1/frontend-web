# 🧩 1-2. 폴더 및 파일 네이밍 컨벤션 (Python 기준)

## 🎯 목적
Python 기반 프로젝트에서 일관된 구조와 가독성을 유지하기 위함.

---

## 📘 기본 규칙
- 폴더명, 파일명 모두 **소문자(snake_case)** 사용  
- **하이픈(-)**, **공백**, **대문자** 사용 금지  
- 의미가 명확한 이름만 사용 (`temp.py`, `test.py` 금지)  
- 테스트 파일은 `test_` 접두사 필수 (`test_auth.py`)

---

## ⚙️ 명명 규칙 요약

| 항목 | 규칙 | 예시 |
|------|------|------|
| **폴더명** | 소문자 | `services`, `routes`, `models` |
| **파일명** | snake_case | `user_service.py`, `auth_router.py` |
| **클래스명** | PascalCase | `UserModel`, `AuthService` |
| **함수 / 변수** | snake_case | `get_user_data()`, `access_token` |
| **상수** | 대문자 + 언더스코어 | `MAX_RETRIES = 3` |
| **테스트 파일** | `test_` prefix | `test_auth.py`, `test_user.py` |

---

## 🚫 피해야 할 예시
| 잘못된 예시 | 이유 |
|--------------|------|
| `User.py` | 클래스와 혼동 |
| `user-service.py` | 하이픈(-)은 import 불가 |
| `Temp.py` | 대문자 혼용 |
| `stuff.py` | 의미 모호 |

---

## ✅ 팀 합의 문장
> “Python 코드는 PEP8을 따르며, 모든 폴더와 파일은 **소문자 + 언더스코어(snake_case)** 규칙을 사용한다.  
> 테스트 파일은 `test_` 접두사를 사용하고, 클래스명은 PascalCase로 정의한다.”
